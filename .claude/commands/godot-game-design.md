## 功能说明

根据用户提供的关键词或主题，调用 godot-game-designer 代理生成创意游戏设计概念和完整的设计文档。适用于新游戏创意构思、游戏机制设计、游戏原型开发等场景。

## 执行步骤

1. **参数获取与验证**：
   - 获取用户输入的关键词或主题参数 $1
   - 如未提供参数，提示用户输入游戏设计的关键词或主题

2. **创意生成阶段**：
   - 调用 @godot-game-designer 代理，基于关键词 $1 进行创意游戏设计
   - 代理将提供多个创意游戏概念，包括：
     - 游戏核心玩法机制
     - 游戏设定和世界观
     - 目标受众和游戏类型
     - 独特卖点和创新要素
     - 初步的系统架构建议

3. **创意选择与细化**：
   - 向用户展示生成的游戏创意选项
   - 用户可选择感兴趣的创意方向，或要求调整和组合
   - 选定创意后，进行详细的设计展开

4. **设计文档生成**：
   - @godot-game-designer 创建完整的游戏设计文档
   - 包含游戏机制、关卡设计、美术风格、音效设计等全面内容
   - 提供具体的技术实现建议和 Godot 开发指导

## 文档组织规范

### 输出路径
```
/docs/01_游戏创意/{两位数字编号}_{游戏名称}.md
```

### 文档结构

生成的游戏设计文档应包含：

- **游戏概述**
  - 游戏名称和类型
  - 核心玩法描述
  - 目标受众分析
  - 独特卖点总结

- **游戏机制**
  - 核心玩法循环
  - 操作控制系统
  - 进程和奖励机制
  - 难度曲线设计

- **游戏设定**
  - 世界观和背景故事
  - 角色设计
  - 环境和场景描述
  - 美术风格指导

### 命名规范
- **文件名**：使用中文命名，如 `01_时空解谜.md`、`02_像素冒险.md`
- **编号规则**：使用两位数字编号，便于文档排序和管理

## 使用示例

```bash
# 基于关键词生成游戏设计
/godot-game-design 时间解谜

# 基于主题生成游戏设计
/godot-game-design 太空探索生存

# 生成特定类型游戏设计
/godot-game-design 多人合作塔防
```

## 设计原则

1. **创新性**：鼓励独特的游戏机制和创意概念
2. **可行性**：考虑技术实现难度和开发成本
3. **用户体验**：注重玩家体验和游戏可玩性
4. **技术适配**：充分利用 Godot 引擎特性和优势
5. **扩展性**：为后续功能扩展和内容更新预留空间
