## 功能说明

根据用户提供的关键词或主题，调用 godot-game-designer 代理生成创意游戏设计概念和完整的设计文档。适用于新游戏创意构思、游戏机制设计等场景。

## 执行步骤

1. **参数获取与验证**：
   - 获取用户输入的关键词或主题参数 $1
   - 如未提供参数，提示用户输入游戏设计的关键词或主题

2. **智能创意生成与Agent协调**：
   - **创意复杂度评估**: 根据关键词 $1 智能选择创意生成策略
     * **简单创意**（单一机制、小型游戏）: godot-game-designer + local-developer组合
     * **中等复杂创意**（多个系统、中型游戏）: godot-game-designer + godot-architect协作
     * **复杂创意**（创新玩法、大型游戏）: 全专家agent协作 + 多模型验证
     * **技术创新项目**: 优先使用local-developer进行技术可行性分析

   - **分层创意生成**:
     * **基础创意概念**: 使用Task工具启动godot-game-designer代理进行创意发散
       - 基于关键词 $1 进行创意发散和概念设计
       - 提供多个创意游戏概念选项，包括核心玩法、世界观、目标受众等
       - 初步分析游戏的技术可行性和开发复杂度
     * **技术可行性分析**: 对于涉及技术实现的创意，使用local-developer(qwen3-coder:30b)进行快速评估
       - 分析创意概念的Godot引擎实现可行性
       - 识别关键的技术挑战和解决方案
       - 提供初步的技术实现路径
     * **架构可行性咨询**: 并行启动godot-architect进行架构层面评估
       - 评估创意概念的技术实现可行性
       - 提供初步的系统架构建议和技术选型
       - 识别潜在的技术挑战和解决方案
     * **市场和技术趋势分析**: 使用tavily-mcp搜索相关游戏案例和技术趋势
       - 分析类似游戏的成功案例和市场表现
       - 了解最新的游戏开发技术和趋势
       - 为创意设计提供技术和市场参考

3. **智能创意优化与选择**：
   - **多维度分析**: 使用sequential-thinking工具深度分析所有创意选项
     * 从技术创新性、市场可行性、开发复杂度等维度评估
     * 识别最具潜力的创意方向
     * 整合不同创意的优点形成优化方案
   - **用户交互选择**: 使用AskUserQuestion向用户展示创意选项
     * 提供详细的创意描述和技术分析
     * 允许用户选择、调整或组合不同创意
     * 收集用户的具体偏好和要求
   - **创意细化**: 基于用户选择进行创意的详细展开和优化

4. **智能设计文档生成与Agent协作**：
   - **文档复杂度评估**: 根据创意复杂度选择文档生成策略
     * **简单游戏设计**: godot-game-designer主导 + local-developer辅助
     * **中等复杂设计**: godot-game-designer + godot-architect协作
     * **复杂系统设计**: 全专家agent协作生成全面文档

   - **分层文档生成**:
     * **创意设计文档**: 使用godot-game-designer创建完整的游戏设计文档
       - 详细的游戏机制、关卡设计、美术风格、音效设计
       - 具体的用户体验设计和交互流程
       - 完整的游戏平衡性和难度曲线设计
     * **技术实现指导**: 启动godot-architect提供技术实现指导
       - 详细的系统架构设计和技术选型
       - Godot引擎特定的实现建议和最佳实践
       - 性能优化策略和扩展性考虑
     * **实现示例生成**: 对于关键技术点，使用local-developer生成实现示例
       - 提供关键机制的伪代码和实现思路
       - 生成具体的算法描述和数据结构设计
       - 为开发团队提供技术参考
     * **文档审查与优化**: 启动相关agent进行专业审查
       - **技术可行性审查**: godot-architect验证技术方案
       - **创意完整性审查**: godot-game-designer确保设计完整性
       - **文档质量审查**: design-doc-writer确保文档标准和可读性
       - **实现示例验证**: local-developer验证技术示例的可行性

> **注意**：不需要输出代码

## 文档组织规范

### 输出路径
```
/docs/01_游戏创意/{两位数字编号}_{游戏名称}.md
```

### 文档结构

生成的游戏设计文档应包含：

- **游戏概述**
  - 游戏名称和类型
  - 核心玩法描述
  - 目标受众分析
  - 独特卖点总结

- **游戏机制**
  - 核心玩法循环
  - 操作控制系统
  - 进程和奖励机制
  - 难度曲线设计

- **游戏设定**
  - 世界观和背景故事
  - 角色设计
  - 环境和场景描述
  - 美术风格指导

### 命名规范
- **文件名**：使用中文命名，如 `01_时空解谜.md`、`02_像素冒险.md`
- **编号规则**：使用两位数字编号，便于文档排序和管理

## 使用示例

```bash
# 基于关键词生成游戏设计
/godot-game-design 时间解谜

# 基于主题生成游戏设计
/godot-game-design 太空探索生存

# 生成特定类型游戏设计
/godot-game-design 多人合作塔防
```

## 设计原则

1. **创新性**：鼓励独特的游戏机制和创意概念
2. **可行性**：考虑技术实现难度和开发成本
3. **用户体验**：注重玩家体验和游戏可玩性
4. **技术适配**：充分利用 Godot 引擎特性和优势
5. **扩展性**：为后续功能扩展和内容更新预留空间
