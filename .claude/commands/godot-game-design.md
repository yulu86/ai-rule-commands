## 执行步骤

### 第一阶段：概念分析与创意生成
1. **关键词解析**: 分析输入关键词的游戏设计潜力
2. **市场调研**: 搜索同类游戏和设计趋势
3. **创意发散**: 基于关键词生成多个游戏概念
4. **概念筛选**: 评估概念的创新性和可行性

### 第二阶段：核心机制设计
1. **核心循环**: 设计游戏的核心玩法循环
2. **机制系统**: 设计支撑核心玩法的游戏系统
3. **平衡设计**: 考虑游戏的平衡性和难度曲线
4. **交互设计**: 设计玩家与游戏的交互方式

### 第三阶段：游戏文档生成
1. **游戏概述**: 编写游戏的高层次描述
2. **详细设计**: 详细阐述游戏机制和系统
3. **用户体验**: 设计UI/UX和反馈系统
4. **技术规格**: 提供Godot实现的技术建议

## Agent 指定

根据不同的设计阶段，系统将自动选择最合适的 agent：

### 概念分析阶段
- **Agent**: `godot-game-designer`
- **职责**: 创意游戏设计概念生成、游戏机制设计
- **专业能力**: 游戏创意构思、玩法创新、用户体验设计

### 技术实现建议
- **Agent**: `godot-architect` (可选)
- **职责**: Godot技术架构设计建议
- **专业能力**: 系统架构、性能优化、技术选型

### 文档整理阶段
- **Agent**: `design-doc-writer` (可选)
- **职责**: 标准化文档格式整理
- **专业能力**: 文档结构化、格式规范化

## 交付件说明

### 主要交付物
1. **游戏概念文档**
   - 游戏名称和类型
   - 核心玩法概述
   - 目标受众分析
   - 独特卖点(USP)

2. **游戏设计文档(GDD)**
   - 完整的游戏机制说明
   - 游戏系统设计
   - 用户体验设计
   - 关卡/内容设计框架

3. **技术实现建议**
   - Godot节点结构建议
   - 关键系统实现方案
   - 性能优化建议
   - 开发里程碑规划

### 文档格式
- **Markdown格式**: 便于版本控制和协作编辑
- **结构化**: 使用标准GDD格式，便于团队协作
- **分层设计**: 从概念概述到详细实现的多层次文档

## 约束条件

### 设计约束
- **创新性**: 确保游戏概念具有独特的创新点
- **可行性**: 考虑Godot引擎的技术限制和实现难度
- **完整性**: 提供足够详细的设计信息以指导开发

### 技术约束
- **Godot兼容性**: 所有设计建议必须基于Godot引擎能力
- **平台适应性**: 考虑目标平台的技术限制
- **性能要求**: 设计必须在目标设备上可流畅运行

### 质量约束
- **可玩性**: 所有机制设计必须以提升游戏性为核心
- **一致性**: 确保游戏各系统间的逻辑一致性
- **扩展性**: 设计时考虑后续内容扩展的可能性

## 使用场景

### 适用场景
- **新项目启动**: 需要完整游戏概念的初始设计
- **创意构思**: 寻找游戏设计的灵感和方向
- **机制设计**: 深化特定游戏系统的设计细节
- **团队协作**: 为开发团队提供清晰的设计文档

### 不适用场景
- **具体代码实现**: 需要使用godot-game-developer命令
- **性能优化**: 需要使用godot-debug命令
- **代码审查**: 需要使用godot-code-reviewer命令

## 最佳实践

### 输入建议
- 使用简洁明确的关键词组合
- 可以包含游戏类型、主题情感、美术风格等元素
- 避免过于宽泛或冲突的概念组合

### 输出使用
- 将生成的GDD作为项目开发的基础文档
- 结合实际开发情况调整设计细节
- 定期回顾和更新设计文档以反映开发进展
