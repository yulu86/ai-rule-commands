# 参数说明

- **功能/模块** (必需): 实现目标，如"用户认证系统"、"支付处理"、"数据分析"等
- **编程语言** (可选): Python、Java、JavaScript/TypeScript、Go、C#、Rust等，默认推荐最适合的语言
- **实现深度** (可选): 核心实现、完整实现、详细实现、生产级实现，默认完整实现

# Agent 与 MCP 工具

## 主要 Agent
- **software-developer** (主导): 高质量代码编写、设计模式应用、性能优化
- **code-reviewer** (质量保证): 代码质量检查、错误识别、最佳实践验证
- **devops-engineer** (部署支持): 部署自动化、监控配置、运维脚本
- **senior-code-reviewer** (深度审查): 深度代码审查、架构验证、性能分析

## MCP 工具
- **sequential-thinking**: 代码架构设计、算法优化、性能调优
- **memory**: 编码规范查询、设计模式库、代码模板
- **context7**: 最新语言特性查询、库文档获取
- **filesystem**: 代码文件创建、编辑、组织

# 交付件

## 主要代码
1. **核心实现代码**: 主要功能实现、核心算法、业务逻辑
2. **数据模型代码**: 数据模型、验证逻辑、数据访问层
3. **测试用例代码**: 单元测试、集成测试、性能测试
4. **配置文件**: 应用配置、环境变量、部署配置
5. **API接口代码**: REST API实现、请求验证、响应格式化
6. **部署脚本**: Docker配置、部署脚本、健康检查

# 约束条件

## 实现约束
- **代码质量**: 确保生成代码的质量和专业性
- **标准合规**: 遵循编程语言规范和最佳实践
- **安全可靠**: 代码实现必须安全可靠，无安全漏洞
- **性能优化**: 考虑代码性能和资源使用效率

## 技术约束
- **语言特性**: 充分利用编程语言的特性和优势
- **框架稳定**: 选择稳定可靠的框架和库版本
- **性能要求**: 满足性能指标和响应时间要求
- **团队技能**: 评估团队的学习成本和技术接受度

# 使用场景

## 适用场景
- **新功能开发**: 业务功能、技术组件、API接口、数据处理
- **系统重构**: 代码优化、技术升级、架构重构、代码清理
- **工具开发**: 开发工具、自动化脚本、测试工具、监控工具
- **原型开发**: 技术验证、概念验证、演示原型、MVP开发

## 不适用场景
- **系统设计**: 不替代系统架构设计工具
- **数据库设计**: 不进行数据库schema设计
- **项目管理**: 不进行项目管理和团队管理
- **运维部署**: 不进行具体的运维操作

# 命令限制说明

此命令专注于专业的代码实现和技术开发指导，不涉及：

- **需求分析**: 使用 `software-requirements-analysis` 命令
- **架构设计**: 使用 `software-architect-design` 命令
- **代码审查**: 使用 `code-reviewer-quality` 命令
- **技术选型**: 使用 `tech-selection-analysis` 命令
- **DevOps支持**: 使用 `devops-engineer` 命令
