# 命令参数说明

- **技术领域** (必需): 指定技术选型的领域，可以是：
  - **前端技术**: 如 "前端框架"、"UI组件库"、"构建工具"等
  - **后端技术**: 如 "后端框架"、"数据库技术"、"消息队列"等
  - **基础设施**: 如 "容器技术"、"云平台"、"监控工具"等
  - **开发工具**: 如 "IDE工具"、"测试框架"、"CI/CD工具"等
  - **新兴技术**: 如 "AI框架"、"区块链平台"、"边缘计算"等

- **候选技术** (可选): 指定需要分析和对比的候选技术
  - **具体技术**: 如 "React vs Vue vs Angular"、"MySQL vs PostgreSQL vs MongoDB"等
  - **技术类别**: 如 "微服务框架"、"NoSQL数据库"、"容器编排"等
  - **自定义列表**: 用户提供的具体技术对比列表
  - **自动推荐**: 根据需求自动推荐候选技术进行对比
  - **默认值**: 如未指定，基于技术领域自动选择主流技术进行对比

- **分析深度** (可选): 指定技术选型分析的详细程度
  - **概要分析**: 高层次的技术对比和关键差异分析
  - **详细分析**: 深入的技术特性分析和优缺点对比
  - **深度分析**: 包含性能测试、成本分析和长期趋势评估
  - **决策级分析**: 包含详细的风险评估、实施策略和ROI分析
  - **默认值**: 如未指定，默认为详细分析

# 执行步骤

## 第一阶段：需求分析与候选技术识别
### 工具使用：sequential-thinking + memory + tavily-mcp

1. **技术需求分析** (使用 sequential-thinking 工具)
   - 深度分析业务需求和技术要求
   - 确定技术选型的关键评估标准
   - 分析技术约束和环境限制条件
   - 制定技术选型的决策框架和权重

2. **市场调研分析** (使用 memory + tavily-mcp 工具)
   - 调研最新的技术趋势和发展方向
   - 收集主流技术的市场份额和使用情况
   - 分析技术社区活跃度和生态支持
   - 研究成功案例和行业最佳实践

3. **候选技术筛选** (使用 sequential-thinking 工具)
   - 基于需求分析筛选候选技术列表
   - 评估技术的成熟度和适用性
   - 分析技术的发展前景和生命周期
   - 制定初步的技术对比矩阵

## 第二阶段：深度技术分析
### 工具使用：software-architect + context7 + sequential-thinking

1. **技术特性对比** (使用 software-architect Agent)
   - 深入对比各候选技术的核心特性
   - 分析技术的功能完整性和扩展性
   - 评估技术的性能表现和资源消耗
   - 设计技术能力对比评估矩阵

2. **生态和社区分析** (使用 context7 工具)
   - 查询各技术的最新文档和版本信息
   - 分析技术社区活跃度和贡献者情况
   - 评估技术生态的完整性和成熟度
   - 研究学习资源和支持渠道

3. **实施可行性评估** (使用 sequential-thinking 工具)
   - 评估技术实施的技术难度和复杂度
   - 分析团队技能匹配度和学习成本
   - 考虑技术集成和迁移的可行性
   - 制定技术实施的资源需求和时间计划

## 第三阶段：风险评估与决策分析
### 工具使用：devops-engineer + senior-code-reviewer + memory

1. **技术风险评估** (使用 devops-engineer Agent)
   - 识别技术选型的潜在风险和问题
   - 分析技术供应商和商业模式风险
   - 评估技术安全性和合规性风险
   - 制定风险缓解和应对策略

2. **成本效益分析** (使用 senior-code-reviewer Agent)
   - 分析技术的总体拥有成本(TCO)
   - 评估技术投资回报率和业务价值
   - 考虑长期维护和升级成本
   - 制定成本控制和优化策略

3. **决策矩阵构建** (使用 memory 工具)
   - 构建多维度的技术决策评估矩阵
   - 设计权重分配和评分标准
   - 收集专家意见和团队反馈
   - 生成量化的技术选型推荐结果

## 第四阶段：选型报告与实施指导
### 工具使用：design-doc-writer + sequential-thinking

1. **选型报告生成** (使用 design-doc-writer Agent)
   - 生成完整的技术选型分析报告
   - 创建技术对比表格和评估结果
   - 编写选型决策说明和理由
   - 制作实施路线图和里程碑规划

2. **实施策略设计** (使用 sequential-thinking 工具)
   - 设计详细的技术实施策略和步骤
   - 制定技术迁移和集成方案
   - 规划团队培训和知识传递计划
   - 建立技术选型的验证和评估机制

3. **长期规划完善** (使用 sequential-thinking 工具)
   - 优化技术的长期发展规划
   - 制定技术演进和升级路径
   - 设计技术风险监控和预警机制
   - 建立持续的技术评估和优化流程

# Agent 与 MCP 工具指定

## 主要执行 Agent

### software-architect (主导 Agent)
- **使用阶段**: 第二阶段技术特性对比、整体技术评估
- **核心职责**: 技术架构分析、技术方案评估、技术决策支持
- **专业能力**: 技术栈评估、架构设计、技术趋势分析、最佳实践应用
- **主要工具**:
  - `sequential-thinking`: 技术分析、决策框架、对比评估
  - `context7`: 最新技术文档查询、特性对比研究
  - `memory`: 技术知识库、最佳实践案例、经验数据

### devops-engineer (风险评估支持 Agent)
- **使用阶段**: 第三阶段技术风险评估、实施可行性分析
- **触发条件**: 需要深度技术风险评估或实施分析时
- **核心职责**: 技术实施风险评估、运维分析、部署可行性评估
- **专业能力**: 基础设施评估、部署复杂度分析、运维成本评估
- **主要工具**:
  - `sequential-thinking`: 风险分析、实施策略、运维评估
  - `memory`: DevOps最佳实践、实施案例库、风险模式

### senior-code-reviewer (成本分析支持 Agent)
- **使用阶段**: 第三阶段成本效益分析、技术债务评估
- **核心职责**: 技术成本分析、ROI评估、长期价值分析
- **专业能力**: 技术经济学、成本建模、投资回报分析
- **主要工具**:
  - `sequential-thinking`: 成本建模、效益分析、ROI计算
  - `memory`: 成本基准、行业数据、评估模型

### design-doc-writer (报告生成支持 Agent)
- **使用阶段**: 第四阶段选型报告生成、文档完善
- **核心职责**: 技术报告撰写、决策文档化、内容组织
- **专业能力**: 技术文档写作、决策分析表达、数据可视化
- **主要工具**:
  - `filesystem`: 报告文档创建、编辑、格式处理
  - `sequential-thinking`: 报告结构设计、内容规划、表达优化
  - `memory`: 报告模板查询、标准格式应用、案例分析

## MCP 工具使用策略

### 1. sequential-thinking 工具
- **使用时机**: 第一阶段需求分析、第二阶段技术对比、第三阶段决策分析、第四阶段策略设计
- **具体用途**:
  - 深度分析技术需求和选型标准
  - 设计合理的技术对比评估框架
  - 分析技术选型的风险和收益
  - 优化技术实施的策略和路线图
- **思考深度**: 根据分析深度要求和技术复杂度调整分析深度

### 2. memory 工具
- **使用时机**: 第一阶段市场调研、第三阶段决策矩阵、全程知识支持
- **具体用途**:
  - 查询技术选型的最佳实践和经验教训
  - 收集相关的技术评估框架和方法论
  - 存储重要的技术决策和评估结果
  - 建立技术选型主题的知识库和案例库
- **记忆策略**: 为每个选型项目建立独立的决策知识图谱

### 3. tavily-mcp 工具
- **使用时机**: 第一阶段市场调研、第二阶段技术趋势分析、第三阶段案例研究
- **具体用途**:
  - 搜索最新的技术趋势和市场动态
  - 收集技术选型的成功案例和失败教训
  - 查询技术社区的最新发展和讨论
  - 获取技术供应商和商业模式的最新信息
- **搜索策略**: 使用技术领域+对比分析+最新趋势进行搜索

### 4. context7 工具
- **使用时机**: 第二阶段技术特性对比、第三阶段深度技术分析
- **具体用途**:
  - 查询候选技术的最新官方文档
  - 获取技术特性的详细对比信息
  - 收集技术API和集成接口信息
  - 查询技术性能基准和测试数据
- **查询策略**: 使用技术名称+特性对比+性能基准进行查询

### 5. filesystem 工具
- **使用时机**: 第四阶段报告生成、全程文档管理
- **具体用途**:
  - 创建和编辑技术选型报告文档
  - 组织评估数据和对比表格
  - 管理相关的调研资料和参考文献
  - 生成不同格式的选型输出文件
- **操作策略**: 遵循文档管理规范和版本控制要求
