# 参数说明

- **技术领域** (必需): 技术选型领域，如"前端框架"、"后端框架"、"数据库技术"等
- **候选技术** (可选): 具体技术对比，如"React vs Vue"、"MySQL vs PostgreSQL"等，默认选择主流技术
- **分析深度** (可选): 概要分析、详细分析、深度分析、决策级分析，默认详细分析

# 执行步骤

**当前技术选型目标**: $ARGUMENTS

## 第一阶段：需求分析与候选技术识别
工具：sequential-thinking + memory + tavily-mcp

1. 技术需求分析
2. 市场调研分析
3. 候选技术筛选

## 第二阶段：深度技术分析
工具：software-architect + context7 + sequential-thinking

1. 技术特性对比
2. 生态和社区分析
3. 实施可行性评估

## 第三阶段：风险评估与决策分析
工具：devops-engineer + senior-code-reviewer + memory

1. 技术风险评估
2. 成本效益分析
3. 决策矩阵构建

## 第四阶段：选型报告与实施指导
工具：design-doc-writer + sequential-thinking

1. 选型报告生成
2. 实施策略设计
3. 长期规划完善

# Agent 与 MCP 工具

## 主要 Agent
- **software-architect** (主导): 技术架构分析、技术方案评估、技术决策支持
- **devops-engineer** (风险评估): 技术实施风险评估、运维分析、部署可行性
- **senior-code-reviewer** (成本分析): 技术成本分析、ROI评估、长期价值分析
- **design-doc-writer** (报告生成): 技术报告撰写、决策文档化

## MCP 工具
- **sequential-thinking**: 技术分析、决策框架、对比评估
- **memory**: 技术知识库、最佳实践、经验数据
- **tavily-mcp**: 市场趋势、技术动态、案例研究
- **context7**: 技术文档查询、特性对比研究

# 交付件

## 主要报告
1. **技术选型分析报告**: 完整的需求分析、技术对比、决策推荐
2. **技术对比矩阵**: 详细的技术特性对比表格、量化评估结果
3. **风险评估报告**: 风险识别、影响分析、缓解策略
4. **实施指导文档**: 实施策略、团队培训、迁移方案

## 报告结构
```markdown
# 技术选型分析报告：[技术领域]

## 1. 项目背景和需求
### 1.1 业务背景和目标
### 1.2 技术需求和约束条件
### 1.3 选型目标和评估标准

## 2. 候选技术概览
### 2.1 技术筛选过程
### 2.2 候选技术清单
### 2.3 技术成熟度评估

## 3. 技术特性对比分析
### 3.1 核心功能对比
### 3.2 性能表现对比
### 3.3 生态系统对比

## 4. 实施可行性评估
### 4.1 技术难度和复杂度
### 4.2 团队技能匹配度
### 4.3 学习成本和时间

## 5. 选型决策和推荐
### 5.1 评估矩阵和权重分配
### 5.2 综合评分和排名
### 5.3 选型推荐和理由
```

# 约束条件

## 分析约束
- **信息准确性**: 确保技术分析的准确性和专业性
- **客观公正**: 保持技术选型的客观性
- **数据驱动**: 基于充分的数据和事实进行分析

## 技术约束
- **信息获取**: 依赖公开的技术文档和市场信息
- **分析深度**: 根据时间和资源限制调整分析深度
- **预测能力**: 技术发展预测存在不确定性

# 使用场景

## 适用场景
- **新项目技术选型**: 新项目、平台选择、工具选型
- **技术升级迁移**: 技术栈升级、架构现代化、云原生转型
- **技术评估调研**: 新技术调研、竞品分析、投资决策
- **团队能力建设**: 技术培训、标准制定、知识管理

## 不适用场景
- **具体实现**: 不进行具体的代码编写和开发
- **系统设计**: 不替代详细的系统架构设计
- **项目管理**: 不进行项目管理和进度控制

# 命令限制说明

此命令专注于技术选型分析和决策支持，不涉及：

- **架构设计**: 使用 `software-architect-design` 命令
- **代码实现**: 使用 `software-developer-guide` 命令
- **需求分析**: 使用 `software-requirements-analysis` 命令
- **DevOps支持**: 使用 `devops-engineer` 命令
