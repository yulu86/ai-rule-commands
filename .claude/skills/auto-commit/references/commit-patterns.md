# Git Commit Message 格式规范

## 基本原则

1. **简洁明了** - 使用动词开头，清晰描述变更内容
2. **控制长度** - 标题不超过72字符，详细信息每行不超过80字符
3. **避免无关信息** - 不包含工具签名、时间戳等冗余信息

## 推荐格式

### 单行格式（推荐用于日常提交）
```
<动词><描述性内容>
```

示例：
- 添加用户认证功能
- 修复登录页面响应式布局
- 更新数据库连接配置
- 删除未使用的工具函数
- 优化商品搜索性能

### 常用动词分类

**添加类**：
- 添加 - 新功能、新文件、新组件
- 新增 - 新增配置、新资源
- 引入 - 新的依赖库、框架
- 实现 - 完整的功能实现

**修改类**：
- 更新 - 配置更新、版本升级
- 修改 - 代码逻辑调整
- 优化 - 性能优化、代码重构
- 调整 - 参数调整、样式微调
- 重构 - 大规模代码重构
- 改进 - 功能改进

**修复类**：
- 修复 - Bug修复
- 解决 - 问题解决
- 完善 - 功能完善

**删除类**：
- 删除 - 移除不需要的代码
- 清理 - 清理冗余代码
- 移除 - 移除过时功能

**文档类**：
- 完善 - 文档补充
- 更新 - 文档更新
- 添加 - 新增文档

## 文件类型描述映射

| 文件类型 | 描述词汇 |
|---------|---------|
| .py | Python文件 |
| .js/.jsx | JavaScript/React组件 |
| .ts/.tsx | TypeScript/React组件 |
| .css/.scss | 样式文件 |
| .html | HTML页面 |
| .md | 文档 |
| .json/.yml/.yaml | 配置文件 |
| .gitignore | Git忽略文件 |
| .env | 环境变量文件 |

## 批量变更的汇总描述

当多个文件变更时，使用汇总性描述：

- **代码变更多**: 代码重构和功能优化
- **配置文件**: 更新项目配置和设置  
- **文档变更**: 完善项目文档和说明
- **混合变更**: 多文件优化和调整

## 特殊情况处理

### 格式化/清理
```
代码格式化和清理
清理未使用的导入
调整代码缩进
```

### 版本升级
```
升级依赖包版本
更新框架到最新版本
```

### 重构
```
重构用户认证模块
优化数据库查询逻辑
简化组件结构
```

## 避免的格式

❌ 包含工具签名：
- "fix: 修复bug 🤖 Generated with Claude Code"
- "feat: 添加功能 Co-Authored-By: Claude"
- "feat: 添加功能 Co-Authored-By: Claude <noreply@anthropic.com> 🤖 Generated with [Claude Code](https://claude.com/claude-code)"

❌ 包含时间戳：
- "update config - 2024-01-01"
- "fix issue at 10:30"

❌ 包含无意义信息：
- "update"
- "fix bug"
- "add file"
