# 类分析模板

## 标准类分析格式

### 类概述
- **类名**: 类的完整名称
- **作用域**: public/private/protected/internal
- **设计模式**: 使用的架构模式
- **职责**: 单一职责原则分析

### 继承关系
- **父类**: 继承的基类和接口
- **子类**: 可能的扩展类
- **多态**: 接口实现和方法重写
- **组合**: 对象组合关系

### 属性分析
- **实例属性**: 对象级别的数据
- **静态属性**: 类级别的共享数据
- **访问控制**: public/private/protected
- **数据类型**: 属性的类型和约束

### 方法分析
- **构造函数**: 对象初始化逻辑
- **实例方法**: 对象行为和操作
- **静态方法**: 类级别的工具方法
- **访问器**: getter/setter 方法

### 设计模式应用
- **创建型模式**: Factory、Builder、Singleton等
- **结构型模式**: Adapter、Decorator、Proxy等
- **行为型模式**: Observer、Strategy、Command等

### 使用场景
- **适用场景**: 什么情况下使用此类
- **优势特点**: 相比其他方案的优点
- **限制条件**: 使用时需要注意的限制
- **最佳实践**: 推荐的使用方式

### 示例代码
```python
# 类使用示例
instance = ClassName(param1, param2)
result = instance.methodName(args)
```
