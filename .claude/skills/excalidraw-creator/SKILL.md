# Excalidraw 图形创建技能

## 核心功能

此技能专门用于使用 Excalidraw 工具创建各种类型的图形和图表，包括但不限于：

- **流程图** - 业务流程、算法流程、工作流程
- **架构图** - 系统架构、软件架构、网络架构  
- **思维导图** - 项目规划、概念整理、头脑风暴
- **快速草图** - UI 设计、概念设计、布局规划
- **组织结构图** - 团队结构、层级关系
- **时序图** - 系统交互、API 调用时序

## 工作流程

### 1. 需求分析
- 理解用户要创建的图形类型和内容
- 确定图形的复杂度和关键元素
- 识别需要的特殊元素（如箭头、连接线、标注等）

### 2. 设计规划
- 设计图形的整体布局和结构
- 确定颜色方案和视觉风格
- 规划元素之间的层次关系

### 3. 元素创建
使用 Excalidraw 工具创建以下基本元素：

**基础图形元素：**
- 矩形（rectangle）- 用于容器、流程步骤
- 椭圆（ellipse）- 用于起止点、强调内容
- 菱形（diamond）- 用于决策点
- 箭头（arrow）- 用于连接、流程方向
- 文本（text）- 用于标注、说明
- 标签（label）- 用于简短说明

**样式和属性：**
- `backgroundColor` - 背景颜色
- `strokeColor` - 边框颜色  
- `strokeWidth` - 边框粗细
- `roughness` - 手绘粗糙度（0-5）
- `opacity` - 透明度（0-1）
- `fontSize` - 字体大小
- `fontFamily` - 字体类型

### 4. 保存和返回
- 自动保存图形到 `.excalidraw/` 目录
- 生成有意义的文件名（如：`用户登录流程图.excalidraw`）
- 返回完整的文件路径供用户访问

## 常用图形模板

### 流程图模板
```
开始(椭圆) → 输入处理(矩形) → 决策判断(菱形) → 处理分支(矩形) → 结束(椭圆)
```

### 架构图模板
```
[前端层] → [API网关] → [服务层] → [数据层]
   ↓           ↓          ↓         ↓
[客户端]    [认证]    [业务逻辑]  [数据库]
```

### 思维导图模板
```
中心主题
├── 主要分支1
│   ├── 子分支1.1
│   └── 子分支1.2
└── 主要分支2
    ├── 子分支2.1
    └── 子分支2.2
```

## 最佳实践

### 布局原则
- **从左到右**：流程图遵循自然的阅读方向
- **从上到下**：层级结构清晰的展示
- **对齐规范**：使用网格对齐保持整齐
- **间距一致**：元素间保持适当的间距

### 颜色使用
- **主色调**：选择1-2种主色保持一致性
- **对比度**：确保文本和背景有足够对比
- **含义区分**：用颜色区分不同类型或状态

### 文本规范
- **简洁明了**：使用简短的表达
- **字体一致**：相同类型元素使用相同字体
- **大小层次**：通过字体大小体现重要性

## 特殊场景处理

### 复杂图形
- 将复杂图形分解为多个小图
- 使用分组功能组织相关元素
- 采用层次化展示结构

### 大型图表
- 分模块创建，逐步完善
- 使用统一的颜色和样式规范
- 保持整体布局的一致性

### 协作需求
- 使用清晰的标注和说明
- 遵循标准的图形符号规范
- 提供必要的图例和注释
