# Godot 2D游戏测试用例设计技能

## 技能概述

`godot-test-designer` 是一个专业的Godot 2D游戏测试用例设计技能，能够根据项目的架构设计文档和详细设计文档，系统化地设计模块化测试策略和测试用例。

## 核心功能

- 🎯 **模块化测试设计**：按照游戏模块维度进行测试用例设计
- 📊 **综合文档输出**：生成包含文字、表格和Mermaid图表的测试设计文档
- 🏗️ **架构驱动**：基于架构设计文档确定测试范围和策略
- 📋 **标准化模板**：提供完整的测试用例和报告模板
- 🔧 **自动化支持**：包含测试用例自动生成脚本

## 使用方法

### 1. 基本使用流程

1. **准备输入文档**
   - 收集项目的架构设计文档
   - 收集详细设计文档
   - 识别核心游戏模块

2. **应用技能设计**
   - 使用本技能分析项目结构
   - 按模块划分测试维度
   - 设计测试策略和用例

3. **生成测试文档**
   - 输出测试策略文档
   - 创建详细测试用例表格
   - 绘制测试架构图

### 2. 技能触发条件

当遇到以下场景时，可以触发此技能：

- 需要为Godot 2D游戏项目设计完整的测试策略
- 需要根据架构文档划分模块化测试用例
- 需要生成标准化的测试设计文档
- 需要设计Godot 2D游戏的功能测试、性能测试等

### 3. 示例使用

```
用户：我有一个Godot 2D平台跳跃游戏，包含玩家控制、敌人AI、关卡系统等模块，请帮我设计测试用例

触发：godot-test-designer 技能
```

## 主要特性

### 🎮 Godot 2D专项测试
- 游戏逻辑模块测试
- 场景管理系统测试
- 输入处理模块测试
- UI/UX交互测试
- 渲染和物理系统测试

- **功能测试**：验证游戏功能正确性

- **集成测试**：模块间交互测试
- **兼容性测试**：多平台适配测试

- 详细测试用例表格
- 测试实施指南
