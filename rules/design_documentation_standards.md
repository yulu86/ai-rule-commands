# design_documentation_standards.md

设计文档规范

## 规范

### 格式
- 文档格式为 markdown

### 内容
- 文档包含文档目的、术语表、系统上下文图、模块划分、核心模型、时序图、状态机、算法/策略、数据存储设计、可靠性设计、安全设计、可观测性设计、测试策略
- 输出的文档中以文字、表格、c4图、流程图、时间线图、时序图、状态图、类图、代码块形式展示
- 代码块上必须包含文字对功能、关键实现说明
- 图形需使用`mermaid`绘制

### 原则
- 文档满足SMART原则
- 设计需要满足SOLID原则
- 设计需要满足KISS原则

## 文档输出路径

- @/docs/{两位数序号}_{主题}/
